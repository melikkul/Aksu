## Summary

Aksu v1.1.0a0 resolves three deficiencies identified in v1.0.0a0:

1. **NeuralBackend CPU-locked** → GPU acceleration stack with CUDA/MPS/XPU auto-detection, bf16, `torch.compile`, batched inference.
2. **TextCleaner missing PDF methods** → `fix_line_breaks()` and `fix_artifacts()` added.
3. **normalize.py missing functions** → `reconstruct_line_breaks`, `fix_pdf_artifacts`, `is_morphologically_valid` implemented and exported.

## Benchmark Status

The NeuralBackend benchmark (`scripts/benchmark_neural_backend.py`) requires the model checkpoint at `models/atomizer_v2/best_model.pt` which is not present on this host.

**To run the benchmark:**
```bash
source .venv-1.1/bin/activate
python scripts/benchmark_neural_backend.py \
    --model-path models/atomizer_v2/best_model.pt \
    --vocab-dir models/vocabs \
    --out audit/benchmark_results/v1.1_neural_backend.json
```

The script will:
1. Measure the baseline (fp32, cpu, no-compile, batch_size=1) first
2. Measure v1.1 configs (compile, CUDA if available)
3. Check CPU acceptance criterion: ≥1.3× baseline AND ≥25 tok/s
4. Write results to `audit/benchmark_results/v1.1_neural_backend.json`

CUDA benchmark: submit to TRUBA akya-cuda for GPU rows. CUDA rows will be null on this CPU-only host.

## Known Limitations / v1.2 Deferred

- **Lexicon coverage**: Zemberek ~100K base forms; inflected surfaces not matched. VH+CP signals carry most decisions. Lemma-aware LEX lookup (root-stripping) deferred to v1.2.
- **Repeated-char collapse**: "→2 chars" conservative rule. Lexicon-aware snap to canonical surface deferred to v1.2.
- **Diacritic restoration**: 11-entry dict stub. Char-LM variant deferred to v1.2.
- **kenlm integration**: `use_lm=True` flag reserved. kenlm scoring activates after `aksu[full]` install in v1.2.
- **CUDA rows in benchmark**: null until TRUBA akya-cuda run.
- **J.2 stable tier**: requires CUDA ≥500 tok/s on akya-cuda. Deferred post-alpha.
