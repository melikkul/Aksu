### Performance Note

The alpha benchmark was run on the TRUBA arf-ui1 login node (CPU-only, 300s CPU time limit). Two constraints apply:

1. **Synthetic checkpoint**: `models/atomizer_v2/best_model.pt` was generated with random weights at production-realistic architecture (`embed_dim=128, hidden_dim=256, num_layers=3, char_vocab_size=106, tag_vocab_size=7807`). The checkpoint carries `"synthetic_random_weights": true` in its metadata. Inference throughput depends on architecture and data movement, not weight values, so the Â§J.1 performance measurement is valid. The stable tier (J.2) will use a trained checkpoint on TRUBA akya-cuda.

2. **torch.compile skipped**: `torch.compile(mode="reduce-overhead")` startup cost (~60-120 CPU seconds) exceeds the login node's 300s CPU time limit. Â§J.1 was verified on batching speedup alone (`batch_size=32` vs `batch_size=1`, both `fp32/cpu/no-compile`). The `torch.compile` path (and full benchmark including CUDA) will be exercised on TRUBA akya-cuda for J.2 stable promotion.

**To reproduce the stable-tier benchmark on akya-cuda:**
```bash
source .venv-1.1/bin/activate
python scripts/benchmark_neural_backend.py \
    --model-path models/atomizer_v2/best_model.pt \
    --vocab-dir models/vocabs \
    --out audit/benchmark_results/v1.1_neural_backend.json
```
